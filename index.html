✨ What SAP doesn't give you

Beyond standard SAP Integration Suite.

Capabilities CPI Pulse adds on top of SAP Cloud Integration — none of these exist in SAP's own native screens.

Multi-select iFlow filtering with live status counts, clickable sender/receiver, and server-side Custom Status search — across both Cloud and Edge Monitoring.
Bulk deploy, bulk redeploy, bulk undeploy, bulk restart, and bulk trace activation — act on multiple selected artifacts in one go instead of one at a time, across Integration Flows, Value Mappings, Message Mappings, and Script Collections.
Saved filter views in monitoring — save a combination of range, status, iFlow, sender and receiver as a named view and restore it instantly.
Custom coloured tags on iFlows — label and group artifacts your own way, independent of SAP packages.
Per-queue JMS thresholds — set individual warning and critical alert levels for each queue, not just one tenant-wide setting.
Consolidated Alerts — failed messages, certificate expiry, queue thresholds and error artifacts in one place, with one click back to where you were.
Draggable health-status button on the SAP page itself — changes colour live based on recent message health, no need to open the panel.
Queue size and overdue-message counts shown together in one table, instead of checking each queue individually.
Certificate expiry shown as days remaining, colour-coded by urgency.
Idle iFlow detection across 1 hour, 24 hours, 7 days, or 30 days — scan depth adapts automatically to your tenant's traffic volume.
All packages and their artifacts in one browsable, searchable page — instead of opening each package individually.
Undeploy history — a running record of what was undeployed and when, kept automatically.
Message Journey — trace a single message end-to-end by Correlation ID, Message ID, or Custom Status from one screen.
Built-in test cases with full payload comparison for both Sync and Async interfaces, showing exactly what passed, failed, or differed.
One Dashboard combining message health, certificates, queues, artifacts and locks — a single screen instead of checking each area separately.
Live deployment status marker shown directly next to each artifact's name on SAP's own design page — no need to switch screens.
A Packages button added straight into SAP's design page header — browse every package and artifact with deployment status without leaving the page.
A dedicated Packages page inside CPI Pulse itself — search, filter by type/status/package, export, and save views, without opening SAP's design page.
Custom Header search and filtering — view all custom headers for a message in Message Journey, and filter monitoring by header Name/Value combined with Sender, Receiver, Custom Status and iFlow.
Bulk Activate and Deactivate for JMS queues — start or stop multiple queues in one action, something SAP's own JMS Queue Monitor doesn't offer at all.
Where Used for JMS queues — select any queue and instantly see every artifact that references it.
The extension remembers which page and tab you were last on — Packages environment, Artifact Overview sub-tab — and restores it after a reload, instead of always reopening on Dashboard.

Built-in test automation

Three ways to test an iFlow, none of them Postman.

One shared, encrypted login for every test. Pick the interface check each test case needs — the form adjusts itself.

Sync runs the full check. Switch to Async and the Expected Payload box disappears instantly — only HTTP and MPL status matter. Switch to Match Payload Async and it's a single GET, compared straight against the response body — no MPL involved. If a payload doesn't match, CPI Pulse shows exactly what's missing, extra, or different — not just pass/fail. No endpoint handy? A ready-made utility iFlow ships with the page for exactly this — download it, deploy it, point the test at it. Note: the utility iFlow only reads Data Store PAYLOAD, Entry ID match — your own process needs to write the result there first.

Setup

From install to first health check.

01

Add the extension

Install from the Chrome Web Store — also works in Microsoft Edge (Chromium), see below. No account, no sign-up screen.

02

Open any SAP Integration Suite page

A small CPI Pulse button appears at the bottom-right. It's already reading your last hour of messages.

03

Pick a tenant volume profile

A one-time setup screen tunes page sizes and lookback windows to your tenant's traffic. Takes ten seconds.

04

Click the button, anytime

Dashboard, monitoring, artifacts, queues, certificates — all there. Drag the button anywhere; it remembers.
